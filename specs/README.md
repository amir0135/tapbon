# Screen specs

One markdown file per screen, written **before** implementation. 5–10 lines each:

- **Purpose** — one sentence
- **Elements** — what's on the screen, top to bottom
- **States** — empty / loading / error
- **Out of scope** — what this screen deliberately doesn't do

Then run the `screen-from-spec` prompt (`.github/prompts/`) against the spec file.
